# Rubrica de Evaluacion - Week 06

## GitHub Features

Criterios de evaluacion para la semana 06 del bootcamp.

---

## Puntuacion Total: 100 puntos

### 1. GitHub Issues (20 puntos)

| Criterio | Puntos | Descripcion |
|----------|--------|-------------|
| Crear Issues | 5 | Minimo 5 issues creados correctamente |
| Labels | 5 | Uso apropiado de labels (bug, feature, docs) |
| Milestones | 5 | Al menos 1 milestone con issues asignados |
| Descripcion | 5 | Issues con descripcion clara y reproducible |

### 2. Pull Requests (25 puntos)

| Criterio | Puntos | Descripcion |
|----------|--------|-------------|
| Crear PR | 5 | PR creado desde branch con cambios |
| Titulo y descripcion | 5 | Formato claro, referencia a Issue |
| Commits limpios | 5 | Commits atomicos con mensajes claros |
| Branch actualizada | 5 | Sin conflictos con base branch |
| Merge completado | 5 | PR mergeado correctamente |

### 3. Code Reviews (25 puntos)

| Criterio | Puntos | Descripcion |
|----------|--------|-------------|
| Revisar PR | 8 | Revision de al menos 1 PR |
| Comentarios utiles | 7 | Comentarios constructivos en lineas |
| Sugerencias | 5 | Uso de sugerencias de codigo |
| Aprobar/Solicitar cambios | 5 | Decision de review documentada |

### 4. Proyecto Final (30 puntos)

| Criterio | Puntos | Descripcion |
|----------|--------|-------------|
| Fork del proyecto | 5 | Fork correcto del repositorio |
| Issue creado | 5 | Issue documentando la contribucion |
| Pull Request | 8 | PR con cambios hacia upstream |
| Code Review recibido | 6 | Respuesta a feedback del reviewer |
| Documentacion | 6 | README actualizado, proceso documentado |

## Criterios Especificos

### Issues bien documentados

```markdown
## Ejemplo de Issue completo:

**Titulo**: Bug: Login button not responding on mobile

**Descripcion**:
### Descripcion
El boton de login no responde en dispositivos moviles.

### Pasos para reproducir
1. Abrir app en movil
2. Ir a pagina de login
3. Tap en boton "Login"
4. No sucede nada

### Comportamiento esperado
Deberia abrir formulario de login

### Screenshots
[imagen adjunta]

### Ambiente
- Device: iPhone 12
- OS: iOS 15
- Browser: Safari
```

### Pull Request bien estructurado

```markdown
## Ejemplo de PR completo:

**Titulo**: feat: add mobile login support

**Descripcion**:
## Descripcion
Agrega soporte para login en dispositivos moviles.

Closes #42

## Cambios
- Ajustar CSS para responsive
- Agregar touch event handlers
- Tests para mobile

## Tipo de cambio
- [x] Bug fix
- [ ] New feature
- [ ] Breaking change

## Checklist
- [x] Tests pasando
- [x] Documentacion actualizada
- [x] Code review solicitado
```

## Entregables Requeridos

1. **Repository** con:
   - Minimo 5 Issues (variedad de tipos)
   - Al menos 1 PR mergeado
   - Project board configurado

2. **Evidencias** de:
   - Code review realizado
   - Templates creados
   - Workflow de colaboracion

3. **Documentacion**:
   - README del proyecto
   - Proceso de contribucion (CONTRIBUTING.md)
